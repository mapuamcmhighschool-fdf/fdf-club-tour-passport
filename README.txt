FDF CLUB TOUR PASSPORT
First Day Fest 2026–2027
Mapúa Malayan Colleges Mindanao – High School Department

CONTENTS
• index.html — Complete website application
• assets/logos/ — Optimized copies of the 16 uploaded club logos
• BOOTH_ACCESS_CODES.txt — Private code list for booth moderators

MAIN FEATURES
• Student registration: full name, nickname, grade level, section, and batch
• Local browser storage for individual device progress
• 16 password-protected club stickers
• Circular peel-and-stick badge animation
• Live 0/16 progress monitoring
• Club Sticker Rack
• Completion notification for all 16 stickers
• Personalized pass generation at any completion level
• PNG image download
• One-page PDF download
• Mobile-responsive interface
• Reset option for shared or reused devices

HOW STUDENTS USE IT
1. Open the website on the student's own device.
2. Enter the student information and tap “Start My Club Tour.”
3. Visit a club booth and tap “Visit & Unlock.”
4. Hand the device to the booth in-charge.
5. The booth in-charge privately enters the assigned access code.
6. The official circular club sticker is added to the passport.
7. Repeat until the student has visited the desired booths or completed all 16.
8. Generate and download the Club Tour Pass.
9. Students who complete all 16 may present the completed passport at the redemption booth.

PRIVACY AND STORAGE
The website does not use a server or database. Student information and progress are stored only through Local Storage in the browser being used. Other students opening the website on different devices or browsers will not see that student's information.

Important:
• Clearing browser data, using private/incognito mode, changing browsers, or tapping “Reset This Device” will erase the saved progress.
• On a shared device, reset the passport before allowing another student to use it.
• The generated pass should be downloaded before resetting or clearing browser data.

GITHUB PAGES DEPLOYMENT
1. Extract this ZIP package.
2. Upload index.html and the assets folder to the root of the existing GitHub repository.
3. Keep the folder structure exactly as provided.
4. Do NOT publicly upload BOOTH_ACCESS_CODES.txt if students can access the repository. Keep this file offline and distribute the individual codes privately to booth personnel.
5. Commit the changes and wait for GitHub Pages to redeploy.

PDF DOWNLOAD
The PDF feature uses the jsPDF library from a public CDN. An internet connection is required when the page first loads for PDF generation. PNG generation and progress tracking work directly in the browser.
